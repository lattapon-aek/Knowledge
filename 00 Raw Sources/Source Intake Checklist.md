# Source Intake Checklist

Checklist สั้น ๆ สำหรับเอา source ใหม่เข้าระบบ

## ก่อน ingest

- ไฟล์นี้เป็นต้นฉบับจริง ไม่ใช่ note สรุป
- อยู่ในโฟลเดอร์ที่เหมาะ (`Articles`, `Papers`, `Assets`)
- ชื่อไฟล์สื่อความหมายพอ
- ถ้ามีภาพหรือไฟล์แนบ แยกเก็บใน `Assets`

## ตอน ingest

- อ่าน source ให้ครบหรือให้พอสำหรับ scope ที่ต้องการ
- ระบุว่า source นี้เกี่ยวกับหมวดไหนของ wiki
- สร้างหรืออัปเดต note ปลายทางที่เกี่ยวข้อง
- เพิ่ม citations / `source:` ให้ note ที่ถูกแก้ไข

## หลัง ingest

- ตรวจว่ามี MOC หรือ note กลางที่ควรอัปเดตหรือไม่
- ตรวจว่ามี synthesis note ที่ควรเสริมไหม
- ถ้ามีข้อมูลขัดแย้งกับของเดิม ให้ระบุไว้ใน note ที่เกี่ยวข้อง
