# การควบคุมต้นทุน: ใช้ให้คุ้ม

**หลักการ:** เลือก model ให้เหมาะกับงาน ไม่ใช้ตัวเดียวกับทุกงาน

---

## เลือก Model ตามงาน

| Model | เหมาะกับ |
|---|---|
| **Claude Haiku** | งานที่ต้องการความเร็วและความเรียบง่าย |
| **Claude Sonnet** | งานทั่วไปที่ต้อง reasoning และ coding |
| **Claude Opus** | งาน reasoning ซับซ้อนหรือ architecture ที่ยากกว่า |

> ⚠️ ราคา Anthropic ปรับได้ตลอดเวลา — ตรวจสอบราคาล่าสุดที่ [anthropic.com/pricing](https://www.anthropic.com/pricing)

> ℹ️ การเลือก model ให้เล็กพอสำหรับงานที่ทำได้ ช่วยลดต้นทุนได้

---

## ระบุ Model ใน Subagent Frontmatter

```markdown
---
name: lint-agent
description: ตรวจ code style และ format
model: haiku          ← alias ของ model
# หรือ
model: claude-haiku-4-5  ← full ID (ล็อก version เฉพาะเจาะจง)
tools:
  - Read
  - Bash
---
```

> ℹ️ **alias** (`haiku`, `sonnet`, `opus`) → ใช้ชื่อย่อของ model
> ℹ️ **full model ID** → ใช้เมื่อต้องการล็อก version เฉพาะเจาะจง

## ระบุ Model ใน Agent Teams

ตั้งผ่าน **natural language** ตอน spawn:

```text
Create a team with 3 teammates to review this PR.
Use Haiku for each teammate to keep costs low.
```

---

## Token Usage โดยประมาณ

| รูปแบบ | Token usage |
|---|---|
| Single session | 1x |
| Subagents | สูงกว่าเล็กน้อย (สรุปผลกลับ main context) |
| Agent Teams | scale ตามจำนวน teammate — แต่ละตัว = Claude instance แยก |
| Agent Teams (plan mode) | มีช่วง plan/approval ก่อน implementation ซึ่งเพิ่ม turn และ context usage |

> ℹ️ Official docs ระบุว่า Agent Teams ใช้ token “significantly more” เมื่อเทียบกับ session เดียว

---

## แนวทางควบคุมต้นทุน Team

- ใช้ **Sonnet** สำหรับ teammates (balance capability vs cost)
- **ทีมเล็ก** — 3-5 teammates เหมาะกับงานส่วนใหญ่
- **spawn prompt กระชับ** — ทุกอย่างใน prompt บวกเข้า context ตั้งแต่เริ่ม
- **กำหนดงานให้พอดี** — งานเล็กเกินไป overhead จะคุ้มไม่พอ, งานใหญ่เกินไปทีมจะทำงานนานโดยไม่ check-in
