# การควบคุมต้นทุน: ใช้ให้คุ้ม

**หลักการ:** เลือก model ให้เหมาะกับงาน ไม่ใช้ Sonnet ทุกอย่าง

---

## เลือก Model ตามงาน

| Model | เหมาะกับ |
|---|---|
| **Claude Haiku** | lint, สร้าง test, จัดรูปแบบ, งานซ้ำๆ |
| **Claude Sonnet** | logic, architecture, code review |
| **Claude Opus** | complex architectural decisions, multi-step reasoning ซับซ้อนมาก |

> ⚠️ ราคา Anthropic ปรับได้ตลอดเวลา — ตรวจสอบราคาล่าสุดที่ [anthropic.com/pricing](https://www.anthropic.com/pricing)

> ℹ️ ต้นทุนลดได้มากเมื่อเลือก Haiku สำหรับงาน routine แทน Sonnet — ตรวจสอบราคาจริงที่ [anthropic.com/pricing](https://www.anthropic.com/pricing)

## ระบุ Model ใน Subagent Frontmatter

```markdown
---
name: lint-agent
description: ตรวจ code style และ format
model: haiku          ← alias (ชี้ version ล่าสุดเสมอ)
# หรือ
model: claude-haiku-4-5  ← full ID (ล็อก version เฉพาะเจาะจง)
tools:
  - Read
  - Bash
---
```

> ℹ️ **alias** (`haiku`, `sonnet`, `opus`) → ชี้ model เวอร์ชันล่าสุดเสมอ
> ℹ️ **full model ID** → ใช้เมื่อต้องการล็อก version เฉพาะเจาะจง

## ระบุ Model ใน Agent Teams

ตั้งผ่าน **natural language** ตอน spawn:

```text
Create a team with 3 teammates to review this PR.
Use Haiku for each teammate to keep costs low.
```

## Token Usage โดยประมาณ

| รูปแบบ | Token usage |
|---|---|
| Single session | 1x |
| Subagents | สูงกว่าเล็กน้อย (สรุปผลกลับ main context) |
| Agent Teams | scale ตามจำนวน teammate — แต่ละตัว = Claude instance แยก |
| Agent Teams (plan mode) | สูงกว่า Agent Teams ปกติ เพราะ teammate รัน read-only exploration ก่อน |

> ℹ️ Official docs ระบุว่า Agent Teams ใช้ token "significantly more" — ขึ้นอยู่กับขนาดทีมและความซับซ้อนของงาน

## แนวทางควบคุมต้นทุน Team

- ใช้ **Sonnet** สำหรับ teammates (balance capability vs cost)
- **ทีมเล็ก** — 3-5 teammates เหมาะกับงานส่วนใหญ่
- **spawn prompt กระชับ** — ทุกอย่างใน prompt บวกเข้า context ตั้งแต่เริ่ม
- **clean up ทีมหลังเสร็จงาน** — teammate ที่ idle ยังกิน token
- **5-6 tasks ต่อ teammate** — เพียงพอต่อการทำงาน ไม่ context switch มากเกินไป
