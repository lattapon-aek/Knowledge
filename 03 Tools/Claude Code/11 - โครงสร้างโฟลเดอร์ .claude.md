# โครงสร้างโฟลเดอร์ .claude

```
โปรเจกต์/
└── .claude/
    ├── CLAUDE.md          ← ความจำร่วมของทีม (ทุก agent อ่าน)
    ├── settings.json      ← การตั้งค่าและ permissions
    ├── commands/          ← คำสั่งพิเศษที่สร้างเอง (/review, /deploy-check)
    │   ├── review.md
    │   └── deploy-check.md
    └── agents/            ← นิยาม Subagent แต่ละตัว
        ├── security.md
        ├── frontend.md
        ├── backend.md
        └── qa-test.md
```

---

## สร้างโครงสร้างด้วย Terminal

```bash
mkdir -p .claude/agents .claude/commands
touch .claude/CLAUDE.md .claude/settings.json
```

หรือสั่ง Claude โดยตรง:

```
> สร้างโครงสร้างโฟลเดอร์ .claude สำหรับโปรเจกต์นี้พร้อม settings.json, agents/ และ commands/
```

---

## Scope ของ Agents

เมื่อมี agent ชื่อเดียวกันหลายตำแหน่ง — ตำแหน่งที่ priority สูงกว่าจะ override

| Location | Scope | Priority |
|---|---|---|
| Managed settings | Organization-wide (deploy โดย IT) | 1 (สูงสุด) |
| `--agents` CLI flag | เฉพาะ session นี้ (ไม่บันทึกลงดิสก์) | 2 |
| `.claude/agents/` | เฉพาะโปรเจกต์นี้ (commit ลง git ได้) | 3 |
| `~/.claude/agents/` | ทุกโปรเจกต์ในเครื่อง | 4 |
| Plugin's `agents/` directory | ตรงที่ plugin ถูก enable | 5 (ต่ำสุด) |
