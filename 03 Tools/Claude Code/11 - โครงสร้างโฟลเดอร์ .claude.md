# โครงสร้าง `.claude`

```
โปรเจกต์/
└── .claude/
    ├── CLAUDE.md          ← instructions หลักของโปรเจกต์
    ├── settings.json      ← การตั้งค่าและ permissions
    ├── rules/             ← rules ที่ scope ตาม path
    ├── skills/            ← skills ที่ใช้ในโปรเจกต์นี้
    ├── commands/          ← คำสั่งพิเศษที่สร้างเอง (/review, /deploy-check)
    │   ├── review.md
    │   └── deploy-check.md
    └── agents/            ← นิยาม Subagent แต่ละตัว
        ├── security.md
        ├── frontend.md
        ├── backend.md
        └── qa-test.md
```

---

## สิ่งที่ official docs ย้ำ

- `CLAUDE.md`, `settings.json`, `skills`, `subagents`, `rules`, และ auto memory เป็นส่วนของ `.claude` / `~/.claude`
- project files ใน `.claude/` เหมาะกับการ commit ลง git เพื่อแชร์กับทีม
- `~/.claude/` เหมาะกับการตั้งค่า personal ที่ใช้ได้ทุกโปรเจกต์
- ถ้าต้องการดู/จัดการ subagents ให้ใช้ `/agents`

---

## สร้างโครงสร้างด้วย Terminal

```bash
mkdir -p .claude/agents .claude/commands
touch .claude/CLAUDE.md .claude/settings.json
```

หรือสั่ง Claude โดยตรง:

```
> สร้างโครงสร้างโฟลเดอร์ .claude สำหรับโปรเจกต์นี้พร้อม settings.json, agents/ และ commands/
```

---

## Scope ของ Agents

เมื่อมี agent ชื่อเดียวกันหลายตำแหน่ง — ตำแหน่งที่ priority สูงกว่าจะ override

| Location | Scope | Notes |
|---|---|---|
| Managed settings | Organization-wide | Managed definitions take precedence over project and user subagents with the same name |
| `--agents` CLI flag | เฉพาะ session นี้ | Passed as JSON and not saved to disk |
| `.claude/agents/` | เฉพาะโปรเจกต์นี้ | Ideal for codebase-specific subagents |
| `~/.claude/agents/` | ทุกโปรเจกต์ในเครื่อง | Personal subagents available in all projects |
| Plugin subagents | ตาม plugin ที่ติดตั้ง | Appear in `/agents`; some frontmatter fields are ignored for security |
